Oliver Iliffe

I'm a third year Computer Science student at King's College London (I will go somewhere a little more inspiring* next year). I like most of what people call "Computer Science", but there are some things I like more than the rest.

I am particularly interested in the tools we use to build systems that are resilient, secure and fast. Roughly, this is the union of some subset of programming languages and some subset of systems. I am interested in type systems and language features that aid performance or safety, as well as efforts to formally define the operational semantics of unsafe, high-level languages.

I value code that leverages its environment. Our operating system and processor present underutilised capabilities. People seem to like to write code that ignores this as if we're programming on little isolated, managed ecosystems (with a big fence around them). For a subject that wouldn't exist without computers, we actually don't like to think about computers that much! I like the complexity (and benefit!) that thinking about the actual system we're running on brings.
